### Key Architectural Decisions
- **Single generic expert executable** instead of per-expert compilation: 2 compiles (prefill+decode) vs 1,440 (60 experts × 24 layers). Uses `matrix_multiplication` op with placeholder tensors instead of `convolution_2d_1x1` with baked constants.
- **Shared expert on ANE, sparse experts on ANE via dynamic dispatch**: Shared expert weights are baked in (like dense FFN). Sparse expert weights are passed as IOSurface placeholders at runtime.
- **CPU router**: Top-K softmax routing runs on CPU (tiny gate linear layer), only selected experts dispatched to ANE.
- **swiglu_ffn_body returns tuple**: `(output, normalized_input)` — needed for MoE because the normalized hidden state feeds both the shared expert AND the router.

- No per-expert compilation — experts share one pre-compiled graph

### Key Architectural Decisions
- **DeltaNet on CPU**: Qwen 3.5's "linear attention" is Gated DeltaNet (recurrent SSM, NOT traditional linear attention). 18/24 layers use fixed-size [128x128] state matrices. Sequential nature makes ANE compilation complex → CPU for now, ANE FFN still used.

- **72 ANE executables**: 6 full-attn layers × 3 × 2 phases + 18 DeltaNet layers × 1 × 2 phases = 72 (under 119 compile budget)
